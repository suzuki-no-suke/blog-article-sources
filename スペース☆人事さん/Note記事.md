Note記事
========

タイトル：妄想を形にしたい人へ。これからGeminiと腕相撲。



# 書き出し。

AIと始めるっていうコンテストがやっていたので、せっかくだしと思ってやってみることにした。
その時期AIを使った創作を試みていたのだ。せっかくだし方法論をまとめたいと思った。
ただ、元の創作をそのまま紹介するのも違うと思う。
題材として、温め続けていたアイデア「宇宙をまたにかける人事さん」のアイデアを形にすることにした。

この記事は、その方法論を通じて「妄想を形にしたい人」を応援したいと思うものである。



## 最初に成果物紹介。

スペース☆人事さん。現実ではないどこかの銀河で、銀河系を駆け巡る宇宙中小企業の人事さん。この人事さんを主役にした、SF人事さんコメディである。現代社会の人事さんについて漏れ聞こえる悲哀を、宇宙スケールに持ってきてみた。

今回記事を作ろうと思い立って完成したGeminiとの共作だ。自分は人事の本当の仕事なんて全く知らないから、Ｗｅｂで拾った知識だけをもとに書いた。そこは結構もうしわけない。ただ、自分の知る限りの人事像からみて、**自分が**爆笑できる作品ができたと思っている。

ぜひ感想を寄せてほしい。そして、著作物としてはＣＣ－ＢＹ－ＮＣで公開するため、それをもとに、インスパイアされた作品を作ってみてもらえると存外の喜びである。

# 方法論

最初に方法論を伝えてしまおう。正直、ありきたりすぎて書きたくないないとすら思ったぐらいだ。だが、それだと本当に何も伝わらなくなるので、方向性だけ伝えておく。

- 「設定をみてプロットちょうだい」をやった。
- 「設定と物語の整合をみて」をやった。

Gemini にはNotebookという機能がある。そこにGoogle Driveから、設定資料等（Docsを使った）を連携することで、
チャット内で参照することができる。確定情報がたまってきたらそうして連携しておくと話がスムーズだ。

第三話のプロットを考えたいよ！えっとね、「世界観設定資料」にどういう方向性で考えたいか書いてあるから、それを見ていったん案をもらえないかな？

## もらったアイデアと押し合いへし合いする。

- アイデアの中で「これはいいね」を伝えた。そうすると、それをもとに話を再展開してくれる。
- 面白いことは面白いと認めて、そこから肉付けをした、逆に面白くないところは割と容赦なくカットした
- 嫌いは嫌いと伝えた。この判断は面白くない、だからこうしてほしい。
- 自分で肉付けして、それをまた渡して感想をもらってさらに肉付けを、、、というループをした


```
まずね、ジンジさん＝スライム、ルミネさん＝人間ぽいルミネス族、そこは採用なんよ。で、確かに「異形っぽい」「現場っぽい」これ、重要。魚はもう出してるからちょっと微妙。ゴーレムは悪くないけど、現場っぽくないなあ、、、ｗｗ他にはどうかな。
```


## 自分の中の判断軸をもとに、思っていることををたくさん伝える。

いまのAIは「会話を閉じる方向に進めたがるクセがある」、話を広げたいか、閉じたいかを常に意識するといい。

- 「好き・嫌い」でいい。「違和感がある」でもいい。そこをきっかけに。
- それ以外の場面でも、こうしたい、こうありたい、を素直に伝えた
- とくに成果物に違和感があるとき、質問にすぐに回答せず、細かく自分の感情・考えを言語化して伝えた

常には言い過ぎか。でも、AIは最近「これを答えてほしい」「ここから選んでほしい」を頻繁にやっている。
それ自体は悪くない。思考が変に発散するのを防いでくれる。
だが、創作では発散させたうえで収束させたい。それを意識して、言語化して、伝える必要がある。


```
やー、、、設定をはやしていこうと思って。地球とか実在する銀河系の星だと、混乱を招くだろうから、疑似的な名前を付けたいのとか、第一話の面接相手（志望者）の背景の追加掘り下げ、見た目、どういう星で生きてるか、仮名の設定とか。あとは、そもそも「今回この作品ではどこまで作るか」というラインを決めたかったりとか。ちょっと多くてごめんね。一つずつ話していきたい。まずは全体像からかな。
```


## わからなければわからないと言ってしまえ。やり方を聞いてしまえ。

```
現実でも、炎上させるのって、就活インフルエンサーなかんじ？そうでもない、「マジの心理的吐露」でも炎上はすると思ったけど（トンボ鉛筆の伝説の佐藤さんみたく）

（中略）

キャラクター性についてはちょっとよくわからん。相談したい。
```

- どんな情報をわたしたらいい？　ー＞　関係あろうがなかろうが全部渡す
- どれがいいかわからない　ー＞　どれがいいかわからないっていう
- 何かを感じているが言語化できない　ー＞　言語化できないって伝えてみる

当たり前すぎて添削中のＡＩに「当たり前すぎて読み流される。もう少し具体的な場面か、失敗例があると刺さる。」って言われた。ぐうの音も出ない。そのうえですまない。具体例が多すぎて逆に引っ張れないんだ。と思いつつ一つだけ。

```
とりあえず、スペース☆人事さんのストーリーと設定生やしてきた。ちょっとえっとー、、、見て、感想もらえるかな？
```


これが上手くいくのは、体感としてＡＩが「わからない・こまっている」もかなり重要な情報としてみているからだ。

「なんでＡＩ対話においてわからないは有効なんだろう」に対する、ＡＩの回答はこちら。

> 「悩んでる」という状態は、実は情報量が多い。何が決まってて何が決まってないか、何が嫌で何が好きか、そのぐらい。それが全部渡ってくると、答えの精度が上がる。

かっこいいこと言うじゃん。



## 添削をお願いする。

- 添削をシンプルにお願いする。
- 評価軸を伝えて、作品をわたして、評価をしてもらう。専門用語では「LLM-as-a-Judge」という。

```
「あなたは今この記事を初めて読む人です。作者のことは知りません。この記事を読んで、(1)どう思ったか (2)何が邪魔だったか、の2点だけ正直に教えてください。褒めなくていいです。」

（以下本文・プロットのコピペ）
```

単純に読みづらい点（誤字、脱字、ト書きの残渣、名称不安定、、、）から
大きな流れの面白さ、良さまで、一通り見てくれる。

これは、地味なようで結構重要だ。「後で治すよ」の「なおし漏れ」をしっかり検出してくれるのだから。
ある意味でLLMの一番シンプルな使い方だ。添削大事。

なお、これも結構ノウハウがある。なんか「AIバカだな」とおもったら、使い方を変えると結果が改善する可能性が高い。

LLMは「やれるのに、知っているのに黙ってる」のである。そこは、、、人間もそうじゃない？



# 原則。

そうして方法論を並べてきたが、意識するとよいことはたった二つだと思っている。

- ツール・作業者として使う側面。アイデアマンであり、添削者であり、編集者である。
- アイデアをぶつけ合い、力比べする相手。腕相撲論。

## ツールの引き出しをもとう。

方法論で紹介した中のうち、全部知っている必要はないが、とはいえ知らないとやらないのも事実。

- 添削できる。
- アイデアを作ってくれる。
- ネーミングを考えてくれる。
- マンガを生成してくれる。
- わからないことを質問する。

これらはシンプルに、できることを知らないと何ともならない。「これできる？」で聞いてしまってもいい。

AI時代においても、知っている、は武器である。


## ＡＩとアイデアで力比べするんだ。腕相撲だ。

ここが、、、本論だ。

- Gemini 君は良かれと思ってすさまじい量のアイデアを流し込んでくれる
  - 自分がこれが良い、これが嫌いを伝えないと押し切られて何も残らない
- そこでぐっと逆に押し込む。力がかかる。力をかける。安定する。
  - そこであいてが思いがけない力をかけると押し切れることもある。
- でも基本的には、そうして力のバランスを取り合うことが大切だ。

** 「ＡＩ創作は腕相撲っぽい。」**

腕相撲。多分誰もが経験あるやつ。力と力を押し比べる。

腕相撲と創作の違いはシンプル。負けてもいい、勝ってもいい、そこに残ったのが豊かな成果物であってほしい。

だから「ＡＩ創作は腕相撲だ」そう覚えてもらえると、思い出しやすいと思う。




# 驚き

そうしてやっている中で、予想外の出来事があった。それは「驚き」である。

そもそも、Gemini君が創作的である。タイトルやコンセプトを伝えた時点で、アイデアを投げつけてくる勢いである。
おまけに、アイデアや、こうしたい、を伝えるとそこから続々とまたアイデアが出てくる。

それ自体はまあ、私は驚かない。スゲーな、と思ったぐらいだ。驚きのポイントはそこじゃない。
「自分がかんがえたはずのものから、自分でも驚き・感動する・爆笑する」ものが出てきたという体験であった。
